# Ideation — Fanstatsy Foosball

## Brain Dump

### Core Motivation
- Build a data science portfolio showcasing ML models and Python tools (PyTorch)
- Statistically increase chances of winning extended family's fantasy football draft + league (2026-2027 season)
- Create ML pipelines and AI pipelines to get historical and real-time info
- Remove emotion from draft picks — pure data-driven decisions

### The Problem
- No consistent pattern for picking draft picks or setting lineup on ESPN app
- Never going to watch enough football to build organic knowledge that beats someone who does
- Loves data, synthesis, and automation — wants to leverage those strengths instead

### What Success Looks Like (by end of 2026-2027 season)
1. Win the 2026-2027 fantasy football league
2. Run draft simulations fast
3. Automate lineup setting for fantasy football
4. Deployable to other fantasy leagues (NBA, etc.)
5. Publish week-by-week data-driven blog posts with league stats and insights
6. Level up from DE/SWE into data science and ML — hands-on learning through building real models

### Key Capabilities Needed
- Historical player/team data ingestion
- Real-time data feeds during the season
- ML models for player valuation and projection
- Draft simulation engine
- Lineup optimization automation (ESPN integration)
- Blog/report generation from stats

### Personal Strengths to Leverage
- Senior SWE/DE — always worked on big data teams, knows pipelines and systems
- STEM background — loves math, but it's been a while since applying it hands-on
- Python experience (learning PyTorch through this project)
- Love of automation and synthesis
- Has built the infrastructure that feeds models — now wants to build the models themselves

### Non-Negotiable: This Project is a Learning Vehicle
- Every feature built must teach statistics, data science, and ML concepts from the ground up
- Starting from scratch as a data scientist — DE/SWE background is the foundation to build on
- Claude should explain the "why" behind every model, algorithm, and statistical concept as it's implemented
- The goal is not just working code — it's transferable understanding that levels up into an AI/ML engineer skill set
- By project end, should be able to confidently build ML models for any domain, not just fantasy football
- Use cloud platforms (AWS, GCP free tiers) alongside local development to learn ML deployment and MLOps
- Every opportunity to practice cloud-based ML (training, serving, pipelines) should be taken

### Two Modes of Working
1. **Learning mode** — Jupyter notebooks where Claude drafts educational content, user works through it, uses Claude to validate understanding. Refreshing STEM/math foundations and building DS/ML from there.
2. **Building mode** — Claude drives implementation of features that help win the league, but always explains what's being built and why. User reviews and learns from the code.
- These modes alternate freely — learning feeds building, building motivates learning.

### Target Professional Identity
- Currently: Senior SWE/DE on big data teams
- Goal: AI/ML/DE/SWE — full stack from data infrastructure through model development
- Portfolio should demonstrate the complete journey and end-to-end capability
